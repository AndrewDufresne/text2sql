📅 分阶段实施路线

四个阶段，8 周完成。每个阶段均有明确的完成标准与红线约束。

P1 Phase 1 — 最小闭环 (W1-W2) ✅

  • 状态服务 + LiteLLM + Langfuse + Trino + Cube + LangGraph + Chainlit + DataHub
  • 1 个问题 → SQL → 结果 → trace 完整 (单表 client, DeepSeek API)
  • make smoke-trino 返回行 + Langfuse 有 trace

P2 Phase 2 — RAG + 安全 (W3-W4) ✅

  • pgvector + TEI + Presidio + sqlglot + OPA 全量接入
  • 注入拦截率 100% (24 例红队: 20 hostile + 4 legit)
  • 多表 join (client × account × exposure) 端到端跑通
  • 92 单测 + 5 契约测试全绿 · 8 E2E 全绿

P3 Phase 3 — HITL + Eval (W5-W6) ✅

  • Argilla 反馈闭环 (👍/👎/修正 SQL) + Golden Set CI 门禁
  • NL 解释节点 + 出参 PII 脱敏 (Presidio)
  • Argilla schema 完整闭环: record_id=trace_id, 6 fields, 14 metadata, 1 vector
  • 120 unit + contracts 全绿 · Golden Set 11 用例 · 阈值 0.90

P4 Phase 4 — 可观测 + 门户 (W7-W8) ✅

  • LGTM 栈: Prometheus + Grafana + Loki + Tempo + Alertmanager + OTel Collector
  • 6 个 text2sql_* Prometheus 指标 + 5 条 SLO 告警规则
  • Grafana 自动 provisioning 3 datasources + t2sql/ai Dashboard (9 卡片)
  • Portainer CE 容器管理 · Backstage 延后

🔒 安全与合规 (Day 1 强制)

从项目第一天起内置的 9 项安全控制措施。

S1

SQL 越权/破坏防护

sqlglot AST 白名单 + 只读账号 + 强制 LIMIT — 任何 DDL/DML/系统函数一律拒绝

S2

行/列权限

Trino 用户身份穿透 (X-Trino-User) + OPA Rego 策略 — RM/Risk/Compliance/Ops/Finance/Admin 六角色细粒度授权

S3

PII 双向外脱敏

Microsoft Presidio — 入口检测 (score≥0.85 必脱敏) + 出口扫描 (结果列级策略)

S4

Prompt 注入防御

Guardrails 规则引擎 + regex 模式匹配 — 20 hostile + 4 legit 红队用例 100% 拦截

S5

模型/提示版本管控

Langfuse Prompt Registry + Git PR — 每个 prompt 有版本号，每次变更可追溯

S6

唯一出网链路

仅 LiteLLM 一条出网链路至 DeepSeek API；.env 集中管理密钥

S7

全链路审计

Langfuse (7天热) + ClickHouse 归档 + Loki — 每个请求 trace_id 端到端落盘 ≥7年

S8

策略即代码

OPA text2sql package — Rego 策略与业务代码同仓库管理，CI 校验

S9

TDD 强制

七层测试金字塔 — 任何 PR 必须先有失败测试再补实现，无测试 = 不合并

🛠️ 技术栈摘要

核心技术选型与决策记录。

LLM Provider

DeepSeek API (via LiteLLM)

ADR-0002: 零 GPU 依赖，DeepSeek-V3，OpenAI 兼容协议。LiteLLM 统一路由，内置 Langfuse callback 自动记录 token + cost。

Backend

Python 3.11 + FastAPI

异步全链路 (httpx, asyncpg, asyncio)。手写 pipeline (非 langgraph.StateGraph)，Pydantic v2 契约优先。

Frontend

Next.js 15 + React 19

Tailwind CSS 3.4, Lucide React icons, react-markdown。Server-side proxy 到 LangGraph，无 CORS 问题。

Query Engine

Trino 442

单节点 coordinator+worker，PostgreSQL connector。X-Trino-User 身份穿透，资源组配额管理。

Vector DB

pgvector (pg16)

HNSW 索引，cosine 距离。schema_chunks 表存储表/列描述嵌入，支持过滤检索。

Observability

LGTM Stack

Prometheus (metrics) + Grafana (dashboards) + Loki (logs) + Tempo (traces) + OTel Collector。6 个 text2sql_* 指标, 5 条 SLO 告警。
